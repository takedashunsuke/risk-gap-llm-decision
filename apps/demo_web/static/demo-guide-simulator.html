7）シミュレーターパターン

実装は apps/demo_web/risk_simulator.py の RiskSimulator。地名・エンディングは docs/design/rp_simulation.md を参照してください。

ステップと上限

項目 内容
計画ステップ 画面のスライダー／リセットが max_steps。内部では 10〜70 にクランプ。
絶対上限 MAX_STEPS = 70 まで「次のステップ」で進行可能（既定モードでは計画到達後も続く場合あり）。
API can_advance＝進行可否、can_decide＝判断ボタン可否の目安。

判断トリガと履歴

判断が必要なとき decision_mode は hard_choice。judgment_trigger_reason の例:

理由コード 意味
max_steps 計画ステップに到達したとき。
late_gap_danger 計画の約 85% 以降で Gap 要注意が立ったとき、一度だけ判断を促す。改善で解除。

履歴は judgment_events。再現性用に rng_seed がスナップショットに含まれます。

判断後の終了（既定モード）

legacy_decision == false（既定）では、判断直後に必ず終了するとは限りません。計画超過後の advance で帰結が決まります。

legacy_decision == true は従来の「判断ですぐエンディング」。デモのリセット API では切り替えず、コードで new_simulator(..., legacy_decision=True) が必要です。

引率プリセット（シナリオ初期値）

ドロップダウンの ID とラベルの対応です。リセット時に sim が初期値にマージされます。

ID ラベル（目安）
safety_first 安全重視・慎重
pace_push 行程・締切重視
optimist 楽観・実績過信
weather_watch 天候・視界警戒
