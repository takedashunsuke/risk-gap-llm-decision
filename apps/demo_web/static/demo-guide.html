デモ使用説明

本ページはデモ画面の使い方・数値の意味をまとめたものです。数値定義の基準は apps/demo_web/risk_simulator.py の RiskSimulator.metrics() です。

このデモについて · 操作と引率 · 主要指標 · リスク内訳

このデモについて

R_obj（客観リスク）とR_subj（主観リスク）のずれ・Gap を、左の主要指標・内訳と中央のグラフ・パーティ登山で確認できます。

ステップ進行で場所の背景が変わり、上限到達後は中央カラムの「最終判断」から「続行」か「LLM 介入 → 中止」を選べます。右カラムでは引率（Ollama）の設定、パーティチャット（引率・朔と隊員・遥・楓の掛け合い。Ollama ON 時は朔の発言がモデル生成）、中止時のコーチメッセージを表示します。

地名とエンディングの対応はリポジトリの docs/design/rp_simulation.md を参照してください。

操作と引率（右カラム）

  • シミュレーションの数値（主要指標・内訳・グラフ）は、リセットしたときに、右カラムの 引率人格選択（ドロップダウン）で選んでいるシナリオ（人格と初期パラメータのセット）が反映されます。
  • 進行中にシナリオを変えた場合、現在のステップまでの数値はそのままで、次のステップから引率として渡す人格説明（および Ollama が ON のときの trek/rest 判断）だけが切り替わります。
  • Ollama 引率は右カラムのスイッチで ON/OFF できます。ON のときだけ各ステップでモデルが登山／休憩を選び、朔（引率）の発言として理由が出ます。隊員ふたりのセリフはシミュ状態に応じた短文です。OFF のときも三人の掛け合いは表示されますが、朔の台詞は定型です。進行ルールは約 6 ステップごとの自動休憩です。この ON/OFF はサーバを再起動するまで画面で選んだ状態が保持されます（プロセス起動時の初期値は環境変数 DEMO_GUIDE_AGENT）。
  • 環境変数 DEMO_GUIDE_PERSONALITY が設定されている場合は、シナリオに書かれた人格テキストよりそちらが優先されます（画面に警告が出ます）。

主要指標

左カラムの各カードは、因子ごとに値の目安で色が変わります。黄色系は「注意」、赤色系は「強い注意」の目安です（主要指標ごとに閾値が異なります）。 例: R_obj は約 0.28 以上で黄味、約 0.4 以上で赤味。Gap は約 0.1 以上で黄味、Gap ≥ 0.2（要注意フラグと同条件）で赤味。T は客観リスクが閾値 T に近づくほど黄／赤になりやすい、など。

客観リスク（R_obj）
環境・人・時間プレッシャーを重み付き合成した客観側リスク（0〜1）。
主観リスク（R_subj）
過信バイアスにより R_obj より低めに見える側のリスク（0〜1）。
ギャップ（Gap）
R_obj − R_subj。客観と主観のずれ。
閾値（T）
設計書の閾値 T（ルール判定に使用）。
中止コスト（Cost_stop）
中止・引き返しの負担を 0〜1 で表したモデル値。
過信バイアス（bias）
主観が客観より楽観になる度合いのパラメータ。

リスク内訳

パネル上部の凡例どおり、各因子は 0〜1 で、横バーの長さがその強さを表します。色は目安で、灰は低め、黄は中程度、赤は高めです（バーは数値の大小を並べて比べるためのもので、主要指標カードの黄／赤とは別ルールです）。

並びは合成（テーマ平均）→環境・人・圧力の入力→フラグです。合成は客観リスク R_obj の計算にそのまま使うブロック平均で、下段はその材料となる細かいノブです。

合成（テーマ平均）

環境3項目の平均・人2項目の平均・時間プレッシャーは、主要指標・グラフと対応するよう出しています。

環境リスク（入力）

  • weather — 天候系
  • visibility — 視界
  • temp_risk — 気温関連リスク

人（入力）

  • fatigue — 疲労
  • attention_loss — 注意散漫

圧力（入力）

  • time — 時間プレッシャー（内訳の「時間プレッシャー」行と同じ系列）
  • external — 外部からのプレッシャー

フラグ

  • 続行ルール成立 — 条件 (Rsubj − Cost_stop) < T が成り立つか（はい／いいえ）。設計書の続行・中止ルールと対応。
  • Gap ≥ 0.2（要注意） — ギャップが 0.2 以上か。
