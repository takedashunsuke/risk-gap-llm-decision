1）デモについて

リスクギャップ（客観と主観のずれ）を、数値とロールプレイ画面で体験するデモです。

R_obj（客観リスク）とR_subj（主観リスク）のずれ・Gap を、左の主要指標・客観リスク内訳と中央のグラフ・パーティ登山で確認できます。ヘッダー文言どおり、客観は外部環境の数値、主観はパーティ・引率（人的エージェント）のニュアンスと対応づけて読めます。

ステップ進行で場所の背景が変わります。中央カラムは「最終判断」ではなく判断イベントとして、計画ステップ到達や状態フラグの切替時にモーダルで選択を促します（理由コードは API の judgment_trigger_reason。詳細はシミュレーターパターン）。

判断後もシミュレーションが即終了しない場合があり、絶対上限（最大 70 ステップ）まで進行できます。エンディングと地名の対応は docs/design/rp_simulation.md を参照してください。

数値定義・判断ロジックの詳細は apps/demo_web/risk_simulator.py の RiskSimulator がソース・オブ・トゥルースです。
