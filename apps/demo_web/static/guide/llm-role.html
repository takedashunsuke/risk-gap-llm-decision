7）今回のシミュレーションにおける Ollama/LLM の役割

このデモの設計は「意思決定と説明の分離」です。判断条件は Python、納得と行動変容の補助は LLM が担います。

キーメッセージ

判断はルール、納得はLLMが担う。

Python が客観的な危険度と条件判定を固定し、LLM はその結果を「人が理解して止まれる言葉」に変換します。

人は数値では止まらないが、意味なら止まれる。

役割 担当 内容
リスク計算 Python 客観的な危険度（`R_obj` / `R_subj` / `Gap`）を算出する。
意思決定条件 Python 続行 / 中止の基準、判断イベントの発火、終了条件を決定する。
認識補正（説明） LLM ズレの指摘、引率チャット、サポートメッセージで理解を補助する。

重要なのは、LLM が判定を置き換えることではなく、判定の意味を伝えて行動変容を起こすことです。意思決定のソース・オブ・トゥルースは常に risk_simulator.py です。

将来拡張

  • 振り返りの自動生成（判断ログから改善点を要約）
  • 説明のパーソナライズ（同じ結果を相手に合わせた言葉で提示）
  • 判断比較の提示（続行/中止の意味を並列表示）

ローカル LLM の利点は、プライバシーを保ちながら低遅延で説明品質を高められる点です。将来拡張でも「判定は Python、説明は LLM」の分離を維持すると、再現性と保守性を両立できます。
