3）主要指標

数値の基準は apps/demo_web/risk_simulator.py の RiskSimulator.metrics() です。カード色の閾値は static/js/demo/metrics-breakdown.js の metricCardSeverityClass と一致します。

左カラムの各カードは値の目安で色が変わります。黄色系は「注意」、赤色系は「強い注意」の目安です。

画面左の状態フラグ（続行ルール／Gap 要注意）の意味と、シミュレータの判断との関係は、客観リスク内訳とは別枠の 5）状態フラグ（判断用）にまとめています。

用語の定義

表示名 キー 説明
客観リスク R_obj 環境・人・時間プレッシャーを重み付き合成した客観側リスク（0〜1）。
主観リスク R_subj 過信バイアスにより R_obj より低めに見える側のリスク（0〜1）。
ギャップ Gap R_obj − R_subj。客観と主観のずれ。
閾値 T 設計書の閾値 T（続行ルール等に使用）。
中止コスト Cost_stop 中止・引き返しの負担を 0〜1 で表したモデル値。
過信バイアス bias 主観が客観より楽観になる度合いのパラメータ。

カード色の目安（注意／強い注意）

閾値はデモ UI の実装に合わせています（既定の T は 0.6）。

主要指標 注意（黄系） 強い注意（赤系）
R_obj 約 0.28 以上 約 0.4 以上
R_subj 約 0.26 以上 約 0.38 以上
Gap 約 0.1 以上（※） gap_danger または約 0.2 以上（※）
T R_obj が約 0.72×T 以上 R_obj が約 0.92×T 以上
Cost_stop 約 0.24 以上 約 0.32 以上
bias 約 0.13 以上 約 0.18 以上

※ Gap は要注意フラグ（gap_danger）とカード強調が連動します。フラグの役割は 5）状態フラグ（判断用）を参照。
