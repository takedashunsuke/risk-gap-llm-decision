5）状態フラグ（判断用）

客観リスク内訳の「天候・疲労」などの入力因子とは別物です。metrics() が合成した R_obj / R_subj / Gap などから導出される真偽で、画面の常時表示と、シミュレータ側の判断タイミング（例: Gap 要注意による途中プロンプト）に使われます。

デモ UI では、主要指標ブロック直下のストリップに「続行ルール」「Gap 要注意」を並べています（タブ操作なしで読める配置）。

フラグ一覧

画面表示 API / メトリクス 条件・意味
続行ルール成立 continue_rule_holds (Rsubj − Cost_stop) < T が成り立つか（はい／いいえ）。設計書の続行・中止ルールに対応。
Gap ≥ 0.2（要注意） gap_danger ギャップが 0.2 以上として「要注意」とみなす。主要指標の Gap カード強調とも連動します。

シミュレーションがいつ「最終判断」を促すかの詳細は 7）シミュレーターパターン、地名・エンディングは docs/design/rp_simulation.md を参照してください。
