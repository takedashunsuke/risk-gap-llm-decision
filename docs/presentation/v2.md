## 2. 既存スライドの言い換え（重要）

### 旧表現（避ける）

- LLMは判断しない
- Pythonが決定する

### 新表現（採用）

- **判断は人間（最終選択）とルール（条件判定）が担う**
- **LLMは、判断の前提を言語化して認識を補正する**

---

## 3. デモとの対応（発表時に言う一文）

- 判断イベントは `max_steps` だけでなく、状態フラグ切替でも発火する
- 中止は即終了、続行はしきい値緩和で継続可能
- それでも判定ロジックは Python が固定し、LLMは説明レイヤに留める
